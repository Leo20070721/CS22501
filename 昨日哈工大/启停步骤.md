# 本地 DeepSeek + Dify 日报助手 启停指南

> 适用环境：Windows 11 + WSL2 (Ubuntu 24.04) + Docker Desktop + Ollama
> 所有数据和工作流均持久化保存，启停不会丢失任何内容。

## 🟢 启动步骤（按顺序执行）

### 1. 启动 Docker Desktop

- 从 Windows 开始菜单打开 **Docker Desktop**。
- 等待系统托盘图标稳定（无感叹号），表示引擎已就绪。

### 2. 启动 WSL2 并运行 Ollama 服务

打开一个 Windows 终端（PowerShell 或终端），输入：

```powershell
wsl
```

进入 Ubuntu 后，启动 Ollama 服务：

```bash
ollama serve &
```

检查模型是否可用：

```bash
ollama list
```
出现 deepseek-r1:14b 即为正常。之后保持此终端窗口打开，或按 Ctrl+D 退出（Ollama 服务会在后台运行）。

3. 启动 Dify 容器组
   仍在 WSL2 终端中，进入 Dify 目录并启动：

```bash
cd ~/dify/docker
docker-compose up -d
```
检查所有服务是否正常：

```bash
docker-compose ps
```
所有容器状态应为 Up 或 running。

4. 访问 Dify
   在浏览器中打开：

```text
http://localhost
```
登录后即可看到你的工作流，随时点击 运行。

🔴 终止步骤（按顺序执行，释放所有资源）

1. 停止 Dify 容器组
   打开 WSL2 终端（若已关闭则先 wsl），执行：

```bash
cd ~/dify/docker
docker-compose down
```
这会停止并移除容器，但工作流和数据库数据仍保留在卷中。

2. 退出 Docker Desktop
   在 Windows 系统托盘找到 Docker 图标（鲸鱼）。

右键点击，选择 “Quit Docker Desktop”。

3. 关闭 WSL2 虚拟机
   在 WSL2 终端输入 exit 或按 Ctrl+D 退出会话。

回到 Windows 终端（PowerShell），执行：

```powershell
wsl --shutdown
```
等待几秒钟，WSL2 彻底关闭，释放全部内存和 GPU 显存。

✅ 注意事项
上述命令均不会删除你的部署文件、模型或工作流数据。

下次使用时严格按 启动步骤 依次执行即可恢复全部功能。

如果某次关机前忘记关服务，直接 wsl --shutdown 也能结束一切（Dify 容器下次需要 docker-compose up -d 重新拉起）。
